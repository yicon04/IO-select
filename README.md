IO-select
  实现多线程服务并发操作，使用技术select，与多进程和多线程技术相比，I/O多路复用技术最大优势是系统开销小，系统不必创建/销毁线程，也不必维护这些进程/线程，从而大大减小了系统开销。
I/O多路转接并发
  使用IO多路径转接函数委托被和检测服务器端所有的文件描述符（通信和监听两类），这个检测过程会导致进程/线程的阻塞，如果检测到已就绪的文件描述符阻塞解除，并将这些已就绪文件描述符传出。
  根据类型对传出的所有已就绪文件描述符进行判断，并作出不同处理
监听的文件描述符：和客户端建立连接
此时调用 accept() 是不会导致程序阻塞的，因为监听的文件描述符是已就绪的（有新请求）
